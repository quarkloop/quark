## Overview

This document is the authoritative catalog of the Quark standard library
(`quark/` namespace). It lists all 18 domains, their subdomains, and every
standard library node.

The URI format is defined in [Node URI Specification](./node-uri). Each
entry below follows the format:

```
quark/<domain>/<subdomain>/<node>:<version>
```

<Callout type="info">
  This catalog is a **living document**. New nodes are added as they're
  implemented. A node listed here without a "✅ Implemented" badge is
  planned but not yet available in the runtime.
</Callout>

## 1. `time` — Scheduling

Nodes that produce data on a time-based schedule.

**Subdomains:** `schedule`

### `time/schedule`

| URI | Description |
|-----|-------------|
| `quark/time/schedule/timer:v1` | Emit a `tick` event at a fixed interval. Config: `interval` (e.g., `"1s"`, `"5m"`) |
| `quark/time/schedule/cron:v1` | Emit a `tick` event on a cron schedule. Config: `schedule` (e.g., `"0 * * * *"`) |

## 2. `net` — Network Protocols

Nodes that operate over network protocols — HTTP, gRPC, MQTT, WebSocket, AMQP.

**Subdomains:** `http`, `grpc`, `mqtt`, `websocket`, `amqp`

### `net/http`

| URI | Description |
|-----|-------------|
| `quark/net/http/webhook:v1` | Receive inbound HTTP POST requests and publish them as events. Config: `path`, `method` |
| `quark/net/http/server:v1` | Full HTTP server with route matching. Config: `routes` (array of `{ method, path }`) |
| `quark/net/http/fetch:v1` | Fetch a URL (from the message payload) and emit the response body. Config: `headers`, `timeout` |
| `quark/net/http/call:v1` | Call an external HTTP API and merge the response into the message. Config: `url`, `method`, `headers` |

### `net/grpc`

| URI | Description |
|-----|-------------|
| `quark/net/grpc/server:v1` | gRPC server — receive requests, publish as events. Config: `port`, `proto` |
| `quark/net/grpc/call:v1` | Call a gRPC method and emit the response. Config: `target`, `method` |

### `net/mqtt`

| URI | Description |
|-----|-------------|
| `quark/net/mqtt/sub:v1` | Subscribe to an MQTT topic and publish messages as events. Config: `broker`, `topic` |
| `quark/net/mqtt/pub:v1` | Publish messages to an MQTT topic. Config: `broker`, `topic` |

### `net/websocket`

| URI | Description |
|-----|-------------|
| `quark/net/websocket/server:v1` | WebSocket server — receive messages, publish as events. Config: `path` |
| `quark/net/websocket/client:v1` | Connect to a WebSocket server and publish received messages. Config: `url` |

### `net/amqp`

| URI | Description |
|-----|-------------|
| `quark/net/amqp/consume:v1` | Consume from an AMQP queue. Config: `url`, `queue` |

## 3. `io` — Filesystem and Stream I/O

Nodes that read from or write to the filesystem, and nodes that operate on

**Subdomains:** `file`, `stream`

### `io/file`

| URI | Description |
|-----|-------------|
| `quark/io/file/watch:v1` | Watch a file or directory for changes and emit events. Config: `path`, `pattern` |
| `quark/io/file/tail:v1` | Tail a file (like `tail -f`) and emit new lines. Config: `path` |
| `quark/io/file/write:v1` | Write or append data to a file. Config: `path`, `mode` (`"write"` \| `"append"`) |
| `quark/io/file/rotate:v1` | Write to a file with log rotation. Config: `path`, `maxSize`, `maxFiles` |
| `quark/io/file/read:v1` | Read a file's contents and emit as an event. Config: `path` |

### `io/stream`

| URI | Description |
|-----|-------------|
| `quark/io/stream/stdin:v1` | Read from standard input and emit as events. |
| `quark/io/stream/stdout:v1` | Write message payload to standard output. |

### `data/predicate`

| URI | Description |

## 13. `route` — Flow Control

Nodes that control where messages go without changing their shape.
Stateless — routing decisions are per-message.

**Subdomains:** `flow`, `control`

### `route/flow`

| URI | Description |
|-----|-------------|
| `quark/route/flow/conditional:v1` | Route to different subjects based on content. Config: `rules` (array of `{ when, emit }`) |
| `quark/route/flow/fan-out:v1` | Broadcast to multiple subjects. Config: `events` (array) |
| `quark/route/flow/fan-in:v1` | Merge multiple subjects into one. |
| `quark/route/flow/join:v1` | Join two streams by correlation key. Config: `key`, `timeout` |
| `quark/route/flow/relay:v1` | Forward unchanged (subject rename). |

### `route/control`

| URI | Description |
|-----|-------------|
| `quark/route/control/throttle:v1` | Rate limit — drop or delay excess messages. Config: `rate`, `per` |
| `quark/route/control/sample:v1` | Random sampling — emit 1 in N messages. Config: `rate` (e.g., `0.1`) |
| `quark/route/control/deduplicate:v1` | Drop duplicate messages by key. Config: `key`, `window` |

### `compute/report`

| URI | Description |
|-----|-------------|
| `quark/compute/report/pdf:v1` | Generate a PDF report. Config: `template`, `data` |
