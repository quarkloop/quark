## Context

You are working on the **Quark Platform** — a three-service system built on
Java 21 + Quarkus 3.x (control plane and data plane), Go (Catalog service +
CLI), NATS for messaging, and GraalJS for TypeScript node evaluation. The
project lives at `/home/z/my-project/workspace/` and is managed via a git
repository with conventional commits.

**Architecture (post-v8):**

| Service | Language | Includes GraalJS? | Role |
|---------|----------|-------------------|------|
| Control plane (`server/`) | Java/Native | ❌ No | REST API, deploy orchestration, spawns data-plane processes |
| Catalog (`quark-catalog/`) | Go + SQLite (pure Go, no CGO) | n/a | Metadata store (systems, nodes, events, sources, registry) |
| Data plane (`runtime/`) | Java/Native | ✅ Yes (via `--macro:truffle-svm`) | Executes nodes, hosts GraalJS, runs providers |
| NATS broker | Go (external) | n/a | Message bus for all inter-service communication |

Persistence is delegated to the Catalog (Go + SQLite via
`modernc.org/sqlite`). DuckDB has been removed entirely. The data plane is
spawned on demand by the control plane's `ProcessManager`.

## 1. Install JDK 21 (or use GraalVM JDK 21)

<Callout type="info">
  If GraalVM is already at `/home/z/graalvm-jdk-21.0.11+9.1/`, skip this step —
  GraalVM JDK 21 includes `javac` and `native-image` and is preferred over
  plain OpenJDK for this project (required for native builds).
</Callout>

Otherwise, install OpenJDK 21:

```bash
cd /home/z/my-project
apt-get download openjdk-21-jdk-headless openjdk-21-jre-headless openjdk-21-jre openjdk-21-jdk
mkdir -p /home/z/jdk21
for deb in openjdk-21-jre-headless_*.deb openjdk-21-jre_*.deb openjdk-21-jdk-headless_*.deb openjdk-21-jdk_*.deb; do
    [ -f "$deb" ] && dpkg-deb -x "$deb" /home/z/jdk21/
done
rm -f openjdk-21-*.deb
```

Set environment (add to `~/.bashrc`):

<Tabs items={["GraalVM (preferred)", "OpenJDK (JVM only)"]}>
  <Tab value="GraalVM (preferred)">
    ```bash
    # GraalVM JDK 21 — includes native-image, required for native builds
    export JAVA_HOME=/home/z/graalvm-jdk-21.0.11+9.1
    export PATH=$JAVA_HOME/bin:$PATH
    ```
  </Tab>
  <Tab value="OpenJDK (JVM only)">
    ```bash
    # Plain OpenJDK 21 — JVM mode only, native builds will not work
    export JAVA_HOME=/home/z/jdk21/usr/lib/jvm/java-21-openjdk-amd64
    export PATH=$JAVA_HOME/bin:$PATH
    ```
  </Tab>
</Tabs>

Verify:

```bash
java -version   # must show 21.x
javac -version  # must show 21.x
# If using GraalVM, also verify native-image:

```

## Important Rules

1. **NATS is required** — the platform uses fail-fast, no fallbacks. If NATS is not running, the server refuses to start.
2. **No fallbacks** — see `AGENTS.md` pitfall #6. Code-level silent fallbacks (in-memory bus when NATS is down, file persistence when Catalog fails, default values when config is missing) are forbidden. Throw, don't degrade.
3. **Build tools must be on PATH** — `java`, `javac`, `go`, `nats-server` must all be available. For native builds, `native-image` is also required (GraalVM only).
4. **Maven wrapper** — use `./mvnw` not `mvn` (Maven is not installed system-wide). The wrapper needs `.mvn/wrapper/maven-wrapper.jar` — never delete this directory.
5. **Legacy jar packaging** — `quarkus.package.jar.type=legacy-jar` in `application.properties` (required for GraalJS classloader in the runtime module).
6. **Go build flag** — `GOFLAGS=-trimpath -buildvcs=false` in Makefile (required for VCS stamping).
7. **State root** — set `quark.state.root` system property (or `QUARK_STATE_ROOT` env var) to control where the Catalog stores `catalog.db` and where data-plane logs are written.
8. **Native image: server ≠ runtime** — the server native build must NOT include `--macro:truffle-svm` (keeps it at 76 MB, 3 GB peak RAM). The runtime native build MUST include it (194 MB, 6.5 GB peak RAM). See `AGENTS.md` pitfall #9.
9. **GraalJS enterprise is broken** — use `org.graalvm.js:js-community` (community edition), not `org.graalvm.polyglot:js` (enterprise). The enterprise variant pulls in `truffle-enterprise`, whose `HSPathGen$EndPoint` has a known native-image incompatibility with GraalVM 21.0.11 + Truffle 24.1.x. See `AGENTS.md` pitfall #10.
10. **Runtime jar naming quirk** — Quarkus produces `<finalName>-runner.jar` as the runnable fat jar. With `finalName=quark-runtime-runner`, the runnable jar is `quark-runtime-runner-runner.jar` (double `-runner` suffix). The thin `quark-runtime-runner.jar` has no main manifest and is NOT runnable.

## Cleanup (between sessions)

```bash
# Stop all Quark-related processes
pkill -f "quark-server-0.1.0-SNAPSHOT-runner" 2>/dev/null
pkill -f "quark-runtime-runner-runner" 2>/dev/null
pkill -f "quark-catalog/quark-catalog" 2>/dev/null
pkill nats-server 2>/dev/null

# Remove build artifacts (optional — keeps disk usage low)
cd /home/z/my-project/workspace && find . -type d -name target -prune -exec rm -rf {} + 2>/dev/null
rm -f cli/quarkctl quark-catalog/quark-catalog

# Remove state (CAUTION: deletes catalog.db and all persisted systems)
rm -rf /home/z/my-project/quark-state
```
