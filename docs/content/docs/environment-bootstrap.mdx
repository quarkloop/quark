## Context

You are working on the **Quark Platform** — a three-service system built on
Java 21 + Quarkus 3.17.8 (control plane and data plane), Go 1.25 (Catalog
service + CLI), NATS for messaging, and GraalJS 24.1.2 for TypeScript node
evaluation.

<Callout type="info">
  Throughout this guide, `$PROJECT_ROOT` refers to the directory containing
  the Quark Platform source tree (the folder with `pom.xml`, `Makefile`,
  `core/`, `server/`, `runtime/`, `cli/`, `quark-catalog/`). Set it once:

  ```bash
  export PROJECT_ROOT=/path/to/quark-platform
  cd "$PROJECT_ROOT"
  ```
</Callout>

**Architecture:**

| Service | Language | Includes GraalJS? | Role |
|---------|----------|-------------------|------|
| Control plane (`server/`) | Java/Native | No | REST API, deploy orchestration, spawns data-plane processes |
| Catalog (`quark-catalog/`) | Go 1.25 + SQLite (pure Go, no CGO) | n/a | Metadata store (systems, nodes, events, sources, registry) |
| Data plane (`runtime/`) | Java/Native | Yes (via `--macro:truffle-svm`) | Executes nodes, hosts GraalJS, runs providers |
| NATS broker | Go (external) | n/a | Message bus for all inter-service communication |

**Key versions:**

| Component | Version |
|-----------|---------|
| JDK | 21 (OpenJDK or GraalVM JDK 21) |
| Quarkus | 3.17.8 |
| GraalJS | 24.1.2 (community edition — `org.graalvm.js:js-community`) |
| Go | 1.25 |
| NATS Server | 2.10.25 |
| Maven | 3.9+ (via `./mvnw` wrapper — no system install needed) |

## 1. Install JDK 21

<Callout type="info">
  GraalVM JDK 21 is preferred over plain OpenJDK for this project because it
  includes `native-image` (required for native builds). If you only need JVM
  mode, plain OpenJDK 21 is sufficient.
</Callout>

Install OpenJDK 21 using your system package manager (`apt`, `brew`,
`dnf`, etc.), or download a tarball from
[https://www.graalvm.org/downloads/](https://www.graalvm.org/downloads/) and
extract it to a directory of your choice.

Set environment (add to your shell profile — `~/.bashrc`, `~/.zshrc`, etc.):

<Tabs items={["GraalVM (preferred)", "OpenJDK (JVM only)"]}>
  <Tab value="GraalVM (preferred)">
    ```bash
    # GraalVM JDK 21 — includes native-image, required for native builds
    export JAVA_HOME=/path/to/graalvm-jdk-21
    export PATH=$JAVA_HOME/bin:$PATH
    ```
  </Tab>
  <Tab value="OpenJDK (JVM only)">
    ```bash
    # Plain OpenJDK 21 — JVM mode only, native builds will not work
    export JAVA_HOME=/path/to/jdk-21
    export PATH=$JAVA_HOME/bin:$PATH
    ```
  </Tab>
</Tabs>

Verify:

```bash
java -version   # must show 21.x
javac -version  # must show 21.x
# If using GraalVM, also verify native-image:
native-image --version
```

## 2. Install Go 1.25

Download from [https://go.dev/dl/](https://go.dev/dl/) and extract:

```bash
curl -fsSL https://go.dev/dl/go1.25.0.linux-amd64.tar.gz -o /tmp/go.tar.gz
mkdir -p /path/to/go
tar -C /path/to/go -xzf /tmp/go.tar.gz --strip-components=1
rm /tmp/go.tar.gz
```

Set environment (add to your shell profile):

```bash
export GOROOT=/path/to/go
export GOPATH=$HOME/go
export PATH=$GOROOT/bin:$GOPATH/bin:$PATH
mkdir -p "$GOPATH"
```

Verify:

```bash
go version  # must show go1.25.x
```

## 3. Download NATS Server

```bash
cd /tmp
curl -fsSL https://github.com/nats-io/nats-server/releases/download/v2.10.25/nats-server-v2.10.25-linux-amd64.tar.gz -o nats.tar.gz
tar xzf nats.tar.gz
mkdir -p "$GOPATH/bin"
cp nats-server-v2.10.25-linux-amd64/nats-server "$GOPATH/bin/nats-server"
chmod +x "$GOPATH/bin/nats-server"
rm -rf nats.tar.gz nats-server-v2.10.25-linux-amd64
```

Verify:

```bash
nats-server --version  # must show v2.10.x
```

## 7. Start the Catalog Service (Background)

```bash
cd "$PROJECT_ROOT"
# Pick a state root directory — this is where catalog.db and dataplane-logs/ live
export QUARK_STATE_ROOT="$PROJECT_ROOT/quark-state"
mkdir -p "$QUARK_STATE_ROOT"
./quark-catalog/quark-catalog \
  -nats nats://localhost:4222 \
  -state "$QUARK_STATE_ROOT" > /tmp/catalog.log 2>&1 &
sleep 2
# Verify
grep "Catalog service ready" /tmp/catalog.log
```

## Important Rules

1. **NATS is required** — the platform uses fail-fast, no fallbacks. If NATS is not running, the server refuses to start.
2. **No fallbacks** — code-level silent fallbacks are forbidden. Throw, don't degrade.
3. **Build tools must be on PATH** — `java`, `javac`, `go`, `nats-server` must all be available. For native builds, `native-image` is also required (GraalVM only).
4. **Maven wrapper** — use `./mvnw` not `mvn` (Maven is not installed system-wide). The wrapper needs `.mvn/wrapper/maven-wrapper.jar` — never delete this directory.
5. **Legacy jar packaging** — `quarkus.package.jar.type=legacy-jar` in `application.properties` (required for GraalJS classloader in the runtime module).
6. **Go build flag** — `GOFLAGS=-trimpath -buildvcs=false` in Makefile (required for VCS stamping).
7. **State root** — set `quark.state.root` system property (or `QUARK_STATE_ROOT` env var) to control where the Catalog stores `catalog.db` and where data-plane logs are written.
8. **Native image: server excludes GraalJS** — the server native build must NOT include `--macro:truffle-svm`. The runtime native build MUST include it.
9. **GraalJS community edition** — use `org.graalvm.js:js-community` (community), not `org.graalvm.polyglot:js` (enterprise). The enterprise variant has a native-image incompatibility.
10. **Runtime jar naming** — Quarkus produces `<finalName>-runner.jar`. With `finalName=quark-runtime-runner`, the runnable jar is `quark-runtime-runner-runner.jar` (double `-runner`).

## Cleanup (between sessions)

```bash
# Stop all Quark-related processes
pkill -f "quark-server-0.1.0-SNAPSHOT-runner" 2>/dev/null
pkill -f "quark-runtime-runner-runner" 2>/dev/null
pkill -f "quark-catalog/quark-catalog" 2>/dev/null
pkill nats-server 2>/dev/null

# Remove build artifacts (optional — keeps disk usage low)
cd "$PROJECT_ROOT" && find . -type d -name target -prune -exec rm -rf {} + 2>/dev/null
rm -f cli/quarkctl quark-catalog/quark-catalog

# Remove state (CAUTION: deletes catalog.db and all persisted systems)
rm -rf "$QUARK_STATE_ROOT"
```
